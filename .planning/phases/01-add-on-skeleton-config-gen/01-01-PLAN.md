<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HA user → Supervisor options.json | User-entered config (entity_ids, influx token, URLs) crosses into the add-on as untrusted strings |
| Supervisor → add-on store listing | repository.yaml/config.yaml control what HA displays and validates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Information Disclosure | influx_token field in config.yaml schema | mitigate | Type field as `password?` so Supervisor masks it in the UI and redacts it from debug logs |
| T-1-02 | Tampering | options schema bounds (batch_interval_minutes, nightly_fit_hour) | mitigate | Constrain with `int(1,1440)` / `int(0,23)` so Supervisor rejects out-of-range values before the container starts |
| T-1-03 | Spoofing | repository.yaml url | accept | Single-maintainer personal repo over HTTPS GitHub; no signing infrastructure in v2.0 scope |
| T-1-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan; image installs are handled and audited in plan 03. PNG generators (Pillow/ImageMagick) are build-host-only and do not enter the shipped image |
</threat_model>
