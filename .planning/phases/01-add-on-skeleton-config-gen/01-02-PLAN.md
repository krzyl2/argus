    Create `argus/rootfs/usr/local/bin/gen-entities.py` (shebang `#!/usr/bin/env python3`). Read the options path from argv[1], default `/data/options.json`. Load JSON, take `options.get("entities", [])`. For a non-empty list, build a dict `{"entities": [ {"entity_id": eid, "friendly_name": "", "detectors": [{"name": "hst", "params": {}}]} for eid in entity_ids ]}` and print `yaml.dump(config, default_flow_style=False, allow_unicode=True, sort_keys=False)`. For an empty list, print `entities: []` and exit 0 (the orchestrator's EntitiesConfigLoader.Validate then raises a clear "contains no entities" error at startup — do not raise here).

    Use `yaml.dump` (PyYAML) — never hand-format YAML — so untrusted entity_id strings from options.json are quoted/escaped safely (mitigates YAML injection, research Security Domain). `params: {}` is intentional: EntitiesConfigLoader deserializes the empty dict and HstParams.From applies all defaults. Do not emit numeric params (HstParams.From parses a Dictionary<string,string>; bare YAML numbers would mistype). PyYAML is the only third-party import; plan 03 installs it into the image and the test below documents it as a host dependency.
  </action>
  <verify>

If set → write ARGUS_DETECTOR_ENDPOINT to the value and the ARGUS_TLS_CA/CERT/KEY paths under /data/certs, mark remote mode. Write `ARGUS_LOG_LEVEL` uppercased (DEBUG|INFO|WARNING) for the detector (detector/argus_detector/config.py reads it). <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supervisor → config-gen | options.json values (entity_ids, tokens, URLs) and SUPERVISOR_TOKEN cross into the script as untrusted input |
| config-gen → s6 env / entities.yaml | materialized secrets and the generated YAML feed both processes |
| Supervisor MQTT service API → config-gen | broker credentials fetched via bashio |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-04 | Information Disclosure | SUPERVISOR_TOKEN / MQTT password / influx_token written to s6 env | mitigate | Write with `printf` to `/var/run/s6/container_environment/` (never `echo`, never log the value); influx_token already masked at schema level (plan 01) |
| T-1-05 | Tampering | YAML injection via entity_id strings in gen-entities.py | mitigate | Use `yaml.dump` exclusively — PyYAML quotes/escapes special characters; never string-format YAML |
| T-1-06 | Denial of Service | MQTT service absent → silent broker auth failure later | mitigate | `services: mqtt:need` + `bashio::services.available` guard with `exit 1` (fail loud, SUPV-02) |
| T-1-07 | Tampering | ws:// port-extraction quirk silently breaks HA auth | mitigate | Write explicit `ws://supervisor:80`; verified by the structural grep gate |
| T-1-SC | Tampering | npm/pip/cargo installs | mitigate | No installs in this plan; PyYAML (a top-tier, BSD-licensed PyPI package) is installed in plan 03 and pinned-by-image; gen-entities.py imports only it from third-party |
</threat_model>
