<objective>
Establish the Phase 1 config seam (CFG-01): make `EntitiesConfigLoader.Validate()` log a warning instead of crashing on empty entities, and add the atomic `ConfigWriter` write path that all later phases (Phase 2+ save endpoints, Phase 3 reload) build on.

Purpose: The orchestrator must not crash on first boot with an empty `options.json` (no entities configured yet) — otherwise the Ingress UI can never load to let the user configure entities. The atomic writer must exist from day one so no future code path can introduce a non-atomic write that races with a FileSystemWatcher.

Output: `ConfigWriter.cs` (new), softened `EntitiesConfigLoader.cs`, new `LogEvents.EmptyEntitiesWarning` EventId, and tests covering empty-entities warning + atomic-write behavior. This plan touches NO `Program.cs` or web-host code — it is pure backend config-layer work and runs in parallel with nothing blocking it.
</objective>
