<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HA Supervisor (172.30.32.2) → Kestrel 0.0.0.0:8099 | Server-to-server Ingress proxy; the only intended ingress path |
| Supervisor → `X-Ingress-Path` request header | PathBase is derived from a request header set by the Supervisor |
| browser → static files (wwwroot) | UseStaticFiles serves committed JS/CSS only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Elevation of Privilege | Kestrel listener bound 0.0.0.0:8099 | mitigate | No `ports:` entry in config.yaml — the port is reachable only via the Supervisor Ingress proxy, never from the LAN. Verified by config.yaml review + `ss -tlnp` showing only 8099 (no host port mapping). |
| T-01-05 | Spoofing | X-Ingress-Path header → Request.PathBase | accept | PathBase is derived from a request header. If the port were directly reachable an attacker could spoof X-Ingress-Path, but the port is NOT exposed (T-01-04) and the header only influences URL prefix generation, not auth or data. Documented accepted risk for Phases 1-2. |
| T-01-06 | Spoofing / EoP | No Ingress session validation (validate_session) | accept | Full Ingress auth middleware (validate_session) is deferred to Phase 4 per CONTEXT.md. Interim trust assumption: accept connections from the Supervisor (172.30.32.2) via the Ingress-only path; HA Ingress already authenticates the user session before proxying. Known, accepted risk for Phases 1-2. |
| T-01-07 | Information Disclosure / Tampering | UseStaticFiles serving wwwroot | mitigate | Static files middleware serves only the committed `wwwroot/` tree (htmx.min.js, argus.css); no user-supplied paths, no directory listing, no /data exposure. Placed after UseRouting so it resolves under the correct PathBase. |
| T-01-08 | Tampering | Placeholder HTML reflecting ingressPath into `<base>` | mitigate | HTML-encode the `X-Ingress-Path` value before interpolating into the `<base href>` attribute to prevent attribute-injection in the served HTML. |
</threat_model>
