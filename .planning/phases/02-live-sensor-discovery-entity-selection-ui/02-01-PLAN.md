<objective>
Build the sensor-discovery backend foundation: extend `HaStateDto` to carry `unit_of_measurement`
and `friendly_name` from the HA `attributes` object, and add a thread-safe `IHaSensorRegistry`
singleton that caches the live numeric-sensor snapshot. The registry is populated by the existing
`NetDaemonHaEventSource` from its existing `get_states` call on every connect — no second WebSocket
(ADR-4 / Anti-Pattern 5).

Purpose: Everything the entity-picker UI displays (entity_id, current value, unit, friendly_name,
tracked pill) comes from this registry. This is the read-cache the HTTP endpoints will consume in Wave 2.
Output: Extended `HaStateDto`, `IHaSensorRegistry` + `HaSensorRegistry`, registry wiring in
`NetDaemonHaEventSource`, new LogEvents ids, and unit tests. Contributes to UI-02 (live sensor listing).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HA Supervisor → orchestrator (WebSocket) | HA state payloads (entity_id, state, attributes) are semi-trusted external input parsed by HaWebSocketClient |
| Worker thread → Kestrel threads (registry) | Registry written by the single WS worker, read concurrently by HTTP threads (Wave 2) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | HaStateDto attributes parsing (GetStatesAsync) | mitigate | Use `JsonElement.TryGetProperty` + `GetString()` only; nullable fields tolerate missing/absent attributes; no string interpolation of attribute values here (HTML-encoding happens at the render boundary in plan 03) |
| T-02-02 | Denial of Service | HaSensorRegistry concurrent access | mitigate | Single-writer + volatile immutable-array reference swap — no lock contention, no torn reads; readers always see a complete snapshot |
| T-02-03 | Information Disclosure | Sensor attributes cached in registry | accept | Registry holds only entity_id/value/unit/friendly_name (no secrets/tokens); friendly_name may be Polish user text — displayed as-is (D8). No secret leakage: the get_states payload contains no credentials |
</threat_model>
