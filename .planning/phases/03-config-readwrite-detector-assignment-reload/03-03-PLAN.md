<form_parsing>
<!-- Indexed detector field parsing — 03-RESEARCH.md Q9 + 03-UI-SPEC.md form encoding. -->
POST keys: detectors[{ei}][{di}][name]=hst and detectors[{ei}][{di}][params][{key}]=value.
{ei} = 0-based index of the entity within the SUBMITTED checkbox list (tracked entities only, in DOM order).
Parse with: `^detectors\[(\d+)\]\[(\d+)\]\[(.+?)\](?:\[(.+?)\])?$` over form.Keys → group by (ei,di) into DetectorConfig.
Correlate ei → entity id positionally with form["entities"] (same DOM/render order — OrderBy(EntityId), Phase 2).
Empty detector list for a submitted entity → insert `new DetectorConfig { Name="hst", Params=[] }` (Pitfall 7 / CFG-03 sane default).
</form_parsing>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /api/sensors/save | Untrusted detector name + param values cross into entities.yaml and back into rendered HTML |
| browser → GET /api/detectors/new-entry | Untrusted entity_idx/det_idx query params |
| save handler → entities.yaml | User-controlled values written to the config file the pipeline reloads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Tampering | Detector param values written to entities.yaml | mitigate | Serialize via the single YamlDotNet SerializerBuilder root-dict call (never string-format) — YamlDotNet escapes special chars; matches T-02-08. Values written as strings into DetectorConfig.Params map. |
| T-03-11 | Information Disclosure / XSS | Param + detector-name values echoed into the page | mitigate | WebUtility.HtmlEncode on every detector.Name and param value in BuildDetectorEntry and disclosure rows (stored-XSS defense); tested with a `<script>` param value. |
| T-03-12 | Tampering / Elevation | /api/detectors/new-entry and extended save reachable without auth | mitigate | IsAuthorizedRequest (X-Ingress-Path header OR RemoteIpAddress 172.30.32.2/loopback) guard on both endpoints (T-02-09 carried over); returns 403 otherwise. Full validate_session is Phase 4. |
| T-03-13 | Denial of Service | Empty detector list crashes reload (Validate throws) | mitigate | Save handler inserts a default HST detector for any empty list before writing, so EntitiesConfigLoader.Validate (re-read before Swap) never throws — a bad config cannot crash the running pipeline (Validate-before-Swap). |
| T-03-14 | Tampering | Malformed entity_idx/det_idx query injection | accept | Values are int.Parse'd and only used as name= indices in a server-rendered fragment; no file/DB/topic access. Bad input → parse failure → benign. Full validation Phase 4. |
</threat_model>
