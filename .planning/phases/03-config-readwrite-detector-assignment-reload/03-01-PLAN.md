NEW — Config/LiveEntitiesConfig.cs (namespace Argus.Orchestrator.Config):
```csharp
public interface ILiveEntitiesConfig
{
    EntitiesConfig Get();
    void Swap(EntitiesConfig newConfig);
    event EventHandler? ConfigChanged;
}

public sealed class LiveEntitiesConfig : ILiveEntitiesConfig
{
    private volatile EntitiesConfig _current;
    public LiveEntitiesConfig(EntitiesConfig initial) => _current = initial;
    public EntitiesConfig Get() => _current;
    public void Swap(EntitiesConfig newConfig)
    {
        Interlocked.Exchange(ref _current, newConfig);
        ConfigChanged?.Invoke(this, EventArgs.Empty);
    }
    public event EventHandler? ConfigChanged;
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | DiscoveryPublisher.RetractAsync | mitigate | Iterate only the passed removedEntities set; topic ids derived from server-controlled EntityConfig via UniqueId.Slug (dots→underscores) — never from raw form input. Test asserts no publish for non-removed entities. |
| T-03-02 | Denial of Service | LiveEntitiesConfig.Get (torn read) | mitigate | `volatile` reference + `Interlocked.Exchange` full barrier — reference reads are atomic on 64-bit; Get() never returns a partially-constructed object. Concurrency test asserts no exceptions and no null. |
| T-03-03 | Tampering | LiveEntitiesConfig.Swap ordering | mitigate | ConfigChanged fires strictly AFTER Interlocked.Exchange, so no subscriber can observe the old reference via Get() during handling. Ordering test asserts Assert.Same(newConfig, captured). |
| T-03-04 | Denial of Service | Retraction with malformed/empty detector list | accept | GetDetectorName falls back to "hst" for empty detector lists — no crash; low risk, single-user LAN tool. |
</threat_model>
