<objective>
Add a `FileSystemWatcher`-based hosted service that reloads `entities.yaml` when it is atomically
replaced (by the UI save or an external editor), coalescing rapid events with a 300ms timer-reset
debounce so a single atomic write triggers exactly one reload (SC4, PITFALLS Pitfall 11).

Purpose: External edits and UI saves alike propagate to the running pipeline within seconds, and
double-fire from the temp→rename sequence is eliminated. Invalid external edits are ignored, never
crashing the pipeline.
Output: `ConfigFileWatcherService` BackgroundService, its DI registration, a new LogEvent, and
tests proving debounce coalescing and invalid-edit tolerance.
</objective>
