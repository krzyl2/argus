<objective>
Make the orchestrator fetch MQTT broker credentials live from the HA Supervisor service API on every connect and reconnect, never caching them, so re-provisioning the Mosquitto add-on survives a reconnect without restarting Argus (SUPV-03). Falls back to the existing env-var credentials when SUPERVISOR_TOKEN is absent (docker-compose / remote-detector deployments), preserving v1 behavior.

Purpose: SUPV-03 — "MQTT credentials are re-read on every reconnect, never cached, so broker re-provisioning does not break egress." This is orchestrator-side work because config-gen runs only once at container start; a long-running process must re-fetch from the fast local Supervisor API itself (PITFALLS.md Pitfall 6).
Output: A credential-source abstraction + Supervisor implementation, a per-attempt refetch in MqttConnection, DI wiring, and unit tests using a faked HttpMessageHandler (no live broker or Supervisor needed).
</objective>

Create `SupervisorMqttCredentialSource` implementing it.
