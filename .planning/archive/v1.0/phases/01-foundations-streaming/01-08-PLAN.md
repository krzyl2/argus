Bidi pattern (RESEARCH Pattern 3, D-04) — completion order is LOAD-BEARING (PITFALL 3):
```csharp
using var call = client.ScoreStream(cancellationToken: ct);
var readTask = Task.Run(async () => {
    await foreach (var verdict in call.ResponseStream.ReadAllAsync(ct))
        OnVerdict(verdict);              // → hysteresis → MQTT
});
await foreach (var reading in haEventChannel.ReadAllAsync(ct))
    await call.RequestStream.WriteAsync(ToPoint(reading), ct);
await call.RequestStream.CompleteAsync();   // FIRST
await readTask;                              // THEN — never reverse (deadlock)
```

Hysteresis (RESEARCH Pattern 7, D-11): high=0.7, low=0.3, min_consecutive=3, per-entity overridable.
