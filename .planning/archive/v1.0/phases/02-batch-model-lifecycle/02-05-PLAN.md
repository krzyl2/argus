fit_one(entity_id, detector, values: list[float]) -> None:
  key=(entity_id, detector); lock=_entity_lock(key)
  snapshot current under lock; train copy.deepcopy OUTSIDE lock; atomic swap under lock (MDL-04)
  candidate = copy.deepcopy(current) if current else _create_detector(detector)

score_batch(entity_id, detector, values) -> tuple[list[float], str | None]:
  read model ref under lock; score OUTSIDE lock (returns None error on success)
  if model is None: raise ValueError (cold-start handled in servicer, not here)
  model may be StlDetector (returns tuple) or PyODDetector (returns list)
  normalize: if model.score_batch returns tuple, return it; if list, return (list, None)

has_model(entity_id, detector) -> bool:
  return (entity_id, detector) in self._detectors

register(entity_id, detector, model_obj):
  directly set self._detectors[(entity_id, detector)] = model_obj (no lock needed for atomic assign in CPython; use self._lock for safety)

_create_detector(detector: str) -> object:
  "mad" | "robust_zscore" → PyODDetector()
  "stl" → StlDetector()
  "hst" → EntityDetector()   (existing streaming detector)
  else → raise ValueError

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gRPC request → ScoreBatch | Entity ID and window values arrive from trusted orchestrator on LAN mTLS channel |
| gRPC request → Fit | Window data arrives from trusted orchestrator; fit mutates registry state |
| disk → LoadModel | Model bytes from disk are trusted (same host; single operator) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Tampering | Fit vs ScoreStream concurrent model swap | mitigate | threading.Lock per-(entity_id, detector); train outside lock on deep copy; atomic swap (MDL-04) |
| T-02-05-02 | Denial of Service | Fit blocks ScoreBatch on same entity | mitigate | Training runs outside entity lock; ScoreBatch reads model ref under lock (O(1)) then scores outside |
| T-02-05-03 | Spoofing | LoadModel accepts arbitrary bytes | accept | Single-operator, self-hosted, model bytes come from same host filesystem (MODEL_ROOT) |
| T-02-05-04 | Denial of Service | load_all_into on startup with many models | accept | Number of entities is small (operator home automation); load is sequential, each model KB-sized |
| T-02-05-05 | Repudiation | Fit cold start not logged | mitigate | logger.info("cold start fit") logged with entity_id and detector fields |
| T-02-05-SC | Tampering | joblib/pickle deserialization | accept | Models loaded from operator-controlled MODEL_ROOT on trusted single-operator host |
</threat_model>
